# 1주차 정리 — HTTP 웹의 기초 & URL 구조

## 1.1 Chapter 01 — HTTP 웹의 기초

### 웹 리소스란?

웹에서 접근 가능한 **모든 콘텐츠**를 말한다.

웹 서버는 각 리소스에 **MIME 타입(Content-Type / Accept-Type)** 을 설정해 클라이언트가 데이터를 어떻게 해석할지 알려준다.

```text
Content-Type: application/json
Accept: text/html
```

### URI(Uniform Resource Identifier)

인터넷 상의 **리소스를 식별하는 식별자**.

```text
  https://www.naver.com/special/download.gif
```

구조:

- **https:// → scheme** (어떻게 접근?)
- [\*\*www.naver.com](http://www.naver.com/) → host\*\* (어디에?)
- **/special/download.gif → resource path** (무엇을?)

---

### HTTP 트랜잭션

HTTP 요청(request) + HTTP 응답(response) 으로 이루어진 **하나의 상호작용 단위**.

---

### TCP Connection

HTTP는 기본적으로 **TCP 위에서 동작**한다.

- 최근 QUIC(UDP 기반 프로토콜)이 빠르게 확산 중 → HTTP/3 의 기반 기술.

---

### 기타 HTTP 구성 요소들

| 용어        | 설명                                              |
| ----------- | ------------------------------------------------- |
| **Proxy**   | 클라이언트–서버 사이 중개자                       |
| **Cache**   | 자주 쓰는 리소스를 가까운 곳에 저장해 빠르게 제공 |
| **Gateway** | 다른 애플리케이션과 연결된 특수 웹서버            |
| **Tunnel**  | HTTP 트래픽을 그대로 전달만 하는 중개자           |
| **Agent**   | 자동화된 HTTP 요청을 생성하는 클라이언트 프로그램 |

## 2. Chapter 02 — URL과 리소스

### 2.1 인터넷 리소스 탐색하기

URL은 다음과 같이 구성되며, 클라이언트가 리소스에 **어떻게 / 어디에 / 무엇을** 요청하는지 표현한다.

```text
<스킴>://<서버주소>/<리소스경로>
```

| 구성요소          | 역할                                      |
| ----------------- | ----------------------------------------- |
| **스킴 (scheme)** | HTTP, HTTPS 등 어떻게 접근할지 (프로토콜) |
| **서버 위치**     | 리소스가 어디에 있는지                    |
| **리소스 경로**   | 서버에 존재하는 어떤 리소스인지           |

---

### 2.2 URL 문법

표준 URL 형태

```text
<스킴>://<사용자>:<비밀번호>@<호스트>:<포트>/<경로>;<파라미터>?<질의>#<프래그먼트>
```

(대부분은 host / path / query 정도만 사용)

---

### 2.3 단축 URL과 상대 URL

### 상대 URL → 절대 URL로 변환 알고리즘

#### **기저(Base) URL**

```text
http://www.joes-hardware.com/tools.html
```

#### **상대 URL**

```text
./hammers.html
```

#### **변환 과정**

1. 스킴이 비어 있으므로 기저 URL의 스킴(http)을 상속
2. 호스트와 포트도 기저 URL에서 상속
3. 상대 경로를 합쳐 완전한 절대 URL 생성

```text
http://www.joes-hardware.com/hammers.html
```

---

### 2.3.2 URL 확장

### 1) 호스트명 확장

`naver` 입력 → 자동으로 `www.naver.com` 으로 보정

### 2) 히스토리 확장

과거 방문 기록을 기반으로 자동 완성 제공

---

## 1주차 요약

- HTTP는 웹 리소스를 요청/응답하는 프로토콜이다.
- URL은 리소스에 접근하기 위한 주소이며 **스킴/호스트/경로**로 구성된다.
- 상대 URL은 기저 URL을 기반으로 절대 URL로 확장된다.
- Proxy, Cache, Gateway 등은 HTTP 생태계를 구성하는 중요한 중개 요소이다.
